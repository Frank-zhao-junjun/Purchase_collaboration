## 1. 文档目的与范围

### 1.1 文档目的

本文档定义「白酒供应链数字化管控平台」与 **SAP S/4HANA ERP** 之间集成层的用户需求规格（URS），作为后续技术设计（FDS）和开发实现的依据。

### 1.2 范围

| 范围 | 说明 |
|------|------|
| **纳入范围** | 平台与 SAP S/4HANA 之间的双向数据同步：采购预测、采购订单、要货计划、收货入库、财务结算、付款状态、主数据（供应商/物料） |
| **排除范围** | 与 SRM/寻源系统的对接（US-201~US-209）、与 WMS/QMS 的直接对接（US-309/US-310 的 WMS 侧）、与电子签系统的对接（US-208），这些属于其他集成线 |
| **不替代** | 本集成层不替代现有平台内业务逻辑，仅负责数据搬运、格式转换、状态追踪与异常处理 |

## 3. 背景与目标

### 3.1 现状

当前平台所有"ERP 同步"数据均为 `init_db.py` 预置的种子数据，无真实 ERP 对接。PRD 中 US-301~US-310、US-401~US-405 共计 15 个用户故事标注了"由 ERP 同步"，但实际未实现任何集成代码。

### 3.2 目标

| 目标维度 | 具体目标 | 验收指标 |
|---------|---------|---------|
| **数据打通** | 实现 S/4HANA 核心业务数据自动同步至平台 | 15 个 US 场景的 ERP 数据源覆盖 100% |
| **实时性** | 关键业务数据准实时同步 | 采购订单/收货数据同步延迟 ≤ 5 分钟 |
| **可追溯** | 每条同步记录可追踪、可审计 | 同步日志保留 ≥ 90 天，支持按单据号检索 |
| **容错性** | ERP 不可用时平台业务不中断 | 断线自动重连，离线数据补传成功率 ≥ 99.5% |
| **可维护** | 集成配置可视化管理 | 字段映射、同步频率、开关均可在管理界面配置 |

### 3.3 设计原则

1. **平台不写回 ERP 业务数据** — 平台是协同门户，不做 ERP 的数据录入入口；出站仅回传"供应商协同结果"（如确认/拒绝、ASN、结算单）
2. **入站优先 OData** — S/4HANA 标准 OData 服务优先，RFC/BAPI 仅在 OData 不可用时使用
3. **幂等设计** — 所有同步操作幂等，重复推送不产生副作用
4. **松耦合** — 集成层与业务层通过内部消息总线解耦，ERP 协议变更不影响业务逻辑

### 4.2 数据流方向

| 方向 | 触发方 | 说明 |
|------|--------|------|
| **入站（ERP → 平台）** | ERP 事件推送 或 平台定时拉取 | ERP 中产生新数据/变更，同步到平台供供应商查看 |
| **出站（平台 → ERP）** | 平台事件触发 | 供应商在平台完成操作后，结果回传 ERP |
| **双向查询** | 平台按需调用 | 平台临时查询 ERP 数据（如实时库存、供应商主数据校验） |

## 6. 功能需求 — 入站同步（ERP → 平台）

### INT-01: 采购预测同步

| 项目 | 内容 |
|------|------|
| **关联 US** | US-301 |
| **SAP 数据源** | 计划订单 (Planned Order) / 采购申请 (Purchase Requisition)，事务码 MD04/ME5A |
| **SAP 接口** | OData: `API_PLANNED_ORDERS` / `API_PURCHASEREQ_PROCESS` |
| **同步频率** | 定时拉取，每 4 小时一次 |
| **数据内容** | 物料编号、工厂、计划数量、计划日期、采购组、供应商分配 |
| **触发条件** | ERP 中计划订单状态变更（创建/修改/转为采购订单） |
| **映射逻辑** | SAP 计划订单 → 平台 `PurchaseForecast` 实体 |

**业务规则：**

1. 同一物料+工厂+计划日期的预测数据，以 ERP 最新版本覆盖平台数据
2. ERP 中已删除的计划订单，平台标记为"已取消"，不物理删除
3. 预测数据同步后自动推送通知到分配的供应商（触发 US-302 协同流程）

---

### INT-02: 采购订单同步

| 项目 | 内容 |
|------|------|
| **关联 US** | US-303 |
| **SAP 数据源** | 采购订单 (Purchase Order, EKKO/EKPO) |
| **SAP 接口** | OData: `API_PURCHASEORDER_PROCESS_SRV` |
| **同步频率** | 事件推送（ERP 输出 IDoc ORDERS05）+ 每 15 分钟定时补拉 |
| **数据内容** | 订单号、供应商、工厂、行项目（物料、数量、价格、交货日期）、付款条件、国际贸易条件 |
| **映射逻辑** | SAP PO → 平台 `PurchaseOrder` + `OrderLineItem` |

**业务规则：**

1. 新增订单：ERP 创建并保存后，同步至平台，平台创建对应记录并通知供应商（US-303）
2. 订单同步后状态为"待供应商确认"，等待供应商在平台操作确认/拒绝/异议
3. 同一 SAP 订单号重复同步时执行幂等更新，仅更新变更字段
4. SAP 订单行项目删除时，平台对应行标记"已删除"，不物理删除

### INT-04: 采购订单变更/关闭同步

| 项目 | 内容 |
|------|------|
| **关联 US** | US-304 |
| **SAP 数据源** | 采购订单变更 (PO Change, CDPOS/CDHDR) |
| **SAP 接口** | OData: `API_PURCHASEORDER_PROCESS_SRV` (读变更版本) |
| **同步频率** | 事件推送（IDoc ORDCHG05）+ 每 15 分钟定时补拉 |
| **数据内容** | 变更字段、变更前值、变更后值、变更时间、变更人 |

**业务规则：**

1. 数量变更：如供应商已确认，平台标记"订单已变更，需重新确认"，通知供应商
2. 价格变更：记录价格历史，通知供应商
3. 交货日期变更：重新推送到供应商确认流程
4. 订单关闭（SAP 设置"交货已完成"或"正式删除"标记）：平台标记订单状态为"已关闭"

### INT-05: 要货计划同步

| 项目 | 内容 |
|------|------|
| **关联 US** | US-305 |
| **SAP 数据源** | 计划协议交货计划 (Scheduling Agreement Delivery Schedule, EKEK/EKET) |
| **SAP 接口** | OData: `API_PURCHASING_SCHEDULE_AGREEMENT_SRV` |
| **同步频率** | 定时拉取，每 1 小时 |
| **数据内容** | 计划协议号、供应商、行项目、交货计划行（日期、数量） |

**业务规则：**

3. 计划行数量调整时，通知供应商查看更新

### INT-07: 收货单同步

| 项目 | 内容 |
|------|------|
| **关联 US** | US-310 |
| **SAP 数据源** | 收货凭证 (Goods Receipt, MSEG/MKPF)，事务码 MIGO |
| **SAP 接口** | OData: `API_MATERIAL_DOCUMENT_SRV` |
| **同步频率** | 事件推送 + 每 10 分钟定时补拉 |
| **数据内容** | 收货凭证号、采购订单号、行项目、物料、收货数量、收货日期、库位、批次号、移动类型 |

**业务规则：**

1. 移动类型 101（采购订单收货）为主同步对象
2. 移动类型 102/122（退货/拒收）也需同步，标记为"退货收货"
3. 收货数据同步后自动创建平台的收货单记录，供应商可查看收货结果（US-310-2）
4. 同一收货凭证号+行号幂等，重复推送不产生重复记录

### INT-08: 质检结果同步

| 项目 | 内容 |
|------|------|
| **关联 US** | US-310 |
| **SAP 数据源** | 质检批次结果 (Inspection Lot, QALS/QAMV)，事务码 QA11 |
| **SAP 接口** | OData: `API_INSPECTIONLOT_SRV` |
| **同步频率** | 事件推送 + 每 10 分钟定时补拉 |
| **数据内容** | 检验批号、物料、批次、检验特性、检验结果值、判定结果（合格/不合格/让步接收） |

**业务规则：**

1. 检验批完成判定（UD）后触发同步
2. 判定结果映射：合格 → 平台"质检通过"，不合格 → "质检不通过"，让步接收 → "让步接收"
3. 质检结果同步后关联到对应收货单，供应商可查看验收结果

### INT-09: 已收货未结算明细同步

| 项目 | 内容 |
|------|------|
| **关联 US** | US-401 |
| **SAP 数据源** | 采购订单历史 (EKBE)，筛选条件：移动类型 101 且无对应发票校验 |
| **SAP 接口** | OData: `API_PURCHASEORDER_PROCESS_SRV`（读取 PO History） |
| **同步频率** | 定时拉取，每 1 小时 |
| **数据内容** | 采购订单号、行项目、收货凭证、物料、数量、金额、收货日期、供应商 |

**业务规则：**

1. 筛选"已收货且未发票校验"的记录，作为供应商创建结算单的可选明细
2. 已在平台创建结算单的收货行标记为"已纳入结算"，不重复推送

## 7. 功能需求 — 出站回传（平台 → ERP）

### INT-03: 采购订单确认/拒绝结果回传

| 项目 | 内容 |
|------|------|
| **关联 US** | US-303 |
| **平台触发事件** | 供应商在平台确认/拒绝/提出异议采购订单 |
| **SAP 接口** | RFC: `BAPI_PO_CHANGE`（设置确认状态）或 OData 更新 |
| **回传数据** | 平台订单号、SAP 订单号、供应商确认状态、确认交货日期、拒绝原因、异议说明 |

**业务规则：**

1. 供应商确认订单：回传确认标记到 SAP PO 的供应商确认字段（EKPO-CONFIRMED）
2. 供应商拒绝订单：在 SAP PO 中创建供应商确认记录，状态为"拒绝"，附拒绝原因
3. 供应商提出异议：在 SAP PO 备注字段记录异议内容，通知采购员在 ERP 中处理
4. 回传失败时平台保留待发送队列，自动重试 3 次，仍失败则告警人工介入

### INT-06: 要货计划确认/调整建议回传

| 项目 | 内容 |
|------|------|
| **关联 US** | US-306 |
| **平台触发事件** | 供应商确认要货计划或提交调整建议 |
| **SAP 接口** | OData: `API_PURCHASING_SCHEDULE_AGREEMENT_SRV`（更新确认） |
| **回传数据** | 计划协议号、行项目、计划行号、确认数量、调整建议数量、调整原因 |

**业务规则：**

1. 供应商确认计划：回传确认标记，SAP 计划行标记为"供应商已确认"
2. 供应商提交调整建议：在 SAP 计划行备注中记录调整建议，不直接修改数量，等待采购方在 ERP 中处理
3. 采购方在 ERP 处理后通过 INT-05 回流更新平台

### INT-11: 发票校验回传

| 项目 | 内容 |
|------|------|
| **关联 US** | US-403 |
| **平台触发事件** | 采购方在平台审批通过供应商提交的发票 |
| **SAP 接口** | RFC: `BAPI_INCOMINGINVOICE_CREATE`（MIRO 过账） |
| **回传数据** | 供应商、发票号、发票日期、金额、税额、采购订单号、收货凭证号、行项目明细 |

**业务规则：**

1. 平台发票审批通过后，自动调用 BAPI 在 SAP 中创建发票校验凭证
2. 三单匹配结果（PO/GR/Invoice）随发票一并传入 SAP，作为 MIRO 的参考
3. SAP MIRO 成功后返回发票校验凭证号，平台记录关联
4. SAP MIRO 失败时（如金额不匹配），平台标记发票为"回传失败"，附带 SAP 错误消息，通知采购方人工处理
5. 同一发票号不可重复过账（幂等校验）

## 9. 非功能需求

### 9.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 同步延迟（事件推送） | ≤ 5 分钟 | ERP 事件产生到平台数据可见 |
| 同步延迟（定时拉取） | ≤ 拉取间隔 + 2 分钟 | 如 INT-02 每 15 分钟拉取，最大延迟 17 分钟 |
| 单次同步吞吐 | ≥ 500 条/批 | 批量拉取时的最小处理能力 |
| 出站回传响应 | ≤ 10 秒 | 单条出站请求的端到端延迟 |
| 并发连接 | ≥ 10 并发 | 同时进行的多场景同步连接数 |

### 9.2 可靠性需求

| 指标 | 要求 |
|------|------|
| 数据不丢失率 | ≥ 99.99%（入站数据在 ERP 中存在，必须最终同步到平台） |
| 同步成功率 | ≥ 99.5%（单次同步成功率，不含自动重试后成功的） |
| 系统可用性 | ≥ 99.5%（月度，不含 ERP 维护窗口） |
| 故障恢复时间 | ≤ 30 分钟（集成层自身故障） |

### 9.3 安全需求

| 安全维度 | 要求 |
|---------|------|
| 传输加密 | 所有 ERP 通信走 HTTPS/TLS 1.2+ |
| 认证方式 | OAuth 2.0 Client Credentials（OData）/ SAP 用户名密码 + 系统号（RFC） |
| 凭据存储 | SAP 连接凭据加密存储（AES-256），不在日志/配置明文中出现 |
| 审计日志 | 所有同步操作记录操作人（系统账号）、时间、数据摘要 |
| 数据脱敏 | 日志中的敏感字段（银行账号、税号）脱敏显示 |
| 网络隔离 | ERP 通信走专用网络通道或 VPN，不暴露公网端口 |

### 9.4 可维护性需求

| 维护维度 | 要求 |
|---------|------|
| 配置热更新 | 字段映射、同步频率、开关状态变更无需重启 |
| 日志检索 | 支持按单据号 30 秒内检索到完整同步链路 |
| 健康检查 | 提供健康检查端点，返回各场景连接状态与最近同步时间 |
| 版本兼容 | SAP 接口升级时，通过适配层隔离变更影响 |

## 10. 技术约束与假设

### 10.1 技术约束

| 约束 | 说明 |
|------|------|
| **SAP S/4HANA 版本** | 假设为 S/4HANA 2021 FPS02 或更高（OData v2/v4 支持） |
| **SAP Gateway** | 已激活，OData 服务已发布 |
| **网络通道** | 平台与 ERP 之间存在网络可达性（VPN 或专线） |
| **平台技术栈** | 后端保持 FastAPI + Python 3.12 + SQLAlchemy 2.0 (async) |
| **集成层实现** | 使用 Python 实现，与后端同语言栈，通过内部消息队列解耦 |
| **SAP 连接库** | OData 使用 `httpx`（异步 HTTP 客户端）；RFC 使用 `pyrfc` 或 SAP NWRFC SDK |
| **不引入中间件** | 不依赖 SAP PI/PO、SAP Integration Suite 等中间件，集成层直接对接 ERP |

### 10.2 假设

| 假设 | 影响验证 |
|------|---------|

| 供应商在平台注册后，通过名称/税号与 ERP 供应商主数据手动匹配 | 需要确认匹配规则 |
| 物料主数据编号在 ERP 和平台间使用同一编码体系 | 需要确认编码规则一致性 |

### 10.3 依赖项

| 依赖 | 提供方 | 用途 |
|------|--------|------|
